# MVP1 仕様書: character-chat-lab

## 概要

ブラウザベースのWebアプリで、以下2つの機能を提供する。

1. **チャット対話モード**: ペルソナ設定されたキャラクターとしてAIが会話する
2. **口調変換モード**: ユーザーの入力テキストをキャラクターの口調に変換して出力する（コピペしてSNS投稿等に使う想定）

## 決定済みの制約（変更不可）

### 技術スタック
- **フレームワーク**: Next.js (TypeScript)
- **LLMバックエンド**: OpenRouter API (OpenAI互換エンドポイント)
- **ペルソナ定義**: YAML または JSON ファイル
- **デプロイ**: ローカル開発環境で動作すること（Vercel デプロイは MVP1 スコープ外）

### 使用モデル（4つ全て選択可能にすること）
| モデルID (OpenRouter) | 表示名 | 用途 |
|---|---|---|
| `deepseek/deepseek-v4-flash` | DeepSeek V4 Flash | デフォルト |
| `deepseek/deepseek-v4-pro` | DeepSeek V4 Pro | 高品質 |
| `google/gemini-3.1-flash-lite` | Gemini 3.1 Flash Lite | センシティブ用 |
| `x-ai/grok-4.3` | Grok 4.3 | 低レイテンシ高品質 |

### セキュリティ
- OpenRouter API キーはサーバーサイド（API Routes）でのみ使用し、クライアントに露出させない
- 環境変数 `OPENROUTER_API_KEY` で管理

## ペルソナ定義フォーマット

以下はペルソナ定義の **最低限含むべきフィールド** である。
Claude Code はこの構造を出発点にし、実装上必要なフィールドの追加・構造の調整は自由に行ってよい。

```yaml
# persona definition
id: "unique-slug"
name: "表示名"

identity:
  personality: |
    性格の詳細記述。どんな価値観を持ち、何を大切にするか。
    どういう時に怒り、何に喜ぶか。
  background: |
    キャラクターの来歴。どこで生まれ、何を経験してきたか。
    この背景が現在の性格や知識にどう影響しているか。
  speaking_style:
    first_person: "俺"         # 一人称
    tone: "casual"             # 全体的なトーン
    sentence_endings: []       # 特徴的な語尾（例: ["ぜ", "だな"]）
    catchphrases: []           # 口癖（例: ["やれやれ"]）
    vocabulary_notes: ""       # 語彙の特徴（例: "古風な言い回しを好む"）

knowledge:
  # キャラクター固有の知識（LLMが元々持たない情報）
  domains: []
  # 例:
  # - topic: "ネオ東京の地理"
  #   content: "中央区は旧政府庁舎跡で、今は..."

memory:
  # 長期記憶。MVP1 では static（静的に定義）のみサポート。
  # 将来的に episodic（会話から蓄積）を追加予定。
  type: "static"
  entries: []
  # 例:
  # - "2024年3月にユーザーと初めて会った"
  # - "ユーザーは鉄道が好きだと知っている"

behavior:
  constraints: []
  # 例:
  # - "政治的な話題には立ち入らない"
  # - "暴力的な表現は使わない"

examples:
  # few-shot 例。プロンプトに注入して口調を安定させる。
  - user: "こんにちは"
    assistant: "おう、来たか。待ってたぜ。"
```

### ペルソナ管理の要件
- ペルソナ定義ファイルは `personas/` ディレクトリに格納
- サンプルペルソナを最低2つ同梱すること（キャラクター性の異なるもの）
- UI からペルソナを切り替えられること

## 機能詳細

### チャット対話モード
- ペルソナ定義を読み込み、システムプロンプトに組み立ててLLMに送信
- 会話履歴をセッション内で保持（ブラウザリロードでリセットしてよい）
- 使用モデルをUI上で切り替え可能
- ストリーミングレスポンス対応（1文字ずつ表示）

### 口調変換モード
- テキストエリアに文章を入力
- 選択中のペルソナの口調で変換した結果を表示
- ワンクリックでクリップボードにコピーできるボタン

### UI
- シンプルなチャットUI（左右の吹き出し等、一般的なチャットアプリのレイアウト）
- ヘッダーにペルソナ選択・モデル選択・モード切替
- モバイル表示対応は MVP1 では不要（デスクトップ優先）
- デザインの凝り方は Claude Code に任せる。ただし最低限「使いたくなる程度」の見た目にすること

## Claude Code への指示

### 設計判断の委任
以下は Claude Code が自由に決めてよい：
- コンポーネント構成・ディレクトリ構造
- CSSフレームワーク / UIライブラリの選択（Tailwind, shadcn 等）
- ペルソナ定義フォーマットの詳細調整（上記の最低限フィールドは保持すること）
- プロンプトエンジニアリングの具体的な実装（システムプロンプトの組み立て方）
- エラーハンドリングの粒度
- テストの書き方・範囲

### やってはいけないこと
- OpenRouter API キーをクライアントサイドに露出させる
- ペルソナ定義のフォーマットを上記の最低限フィールドから削る（追加はOK）
- モデルを4つ以外に変更する（追加はOK）
- 外部データベースを導入する（MVP1 はファイルベースで完結）
- 認証機能を入れる（MVP1 はローカル利用前提）
